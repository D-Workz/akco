Ontology Specification Draft

Abstract

The anatomy of knowledge cleaning ontology represents relevant features of knowledge cleaning. It provides a semantic framework for selecting and applying knowledge cleaning approaches for cleaning Knowledge Graphs. The selection supports choosing appropriate approaches given a specific knowledge graph, cleaning sources, and a specific error type. The application algorithms allow for a structured application of the selected approaches. This application orders the approaches to detect and correct the defined error type. The correction process ensures that the error is a true error and that the predicted correction is an actual correction for the error.

  • Knowledge Graph Features
  • Knowledge Cleaning Approaches & Techniques
  • Dimensions of Approaches
  • Background Knowledge
  • Application Algorithms

The ontology answers a central question:

Given a Knowledge Graph, its properties, cleaning sources, and a specific error type, what are the most suitable approaches for detecting and correcting the errors?

Additionally, it supports subordinate questions regarding:

  • Specific error types
  • Approach dimensions
  • Cleaning source compatibility
  • Background knowledge required for each approach

Introduction back to ToC

The Anatomy of Knowledge Cleaning Ontology (AKCO) provides the semantics for knowledge cleaning. At its core, the ontology includes two key components: the knowledge graph and the cleaning approaches. Its primary goal is to enable the selection and application of appropriate approaches for detecting and correcting errors within a knowledge graph.

Error Representation

Knowledge cleaning needs to target different errors. These errors are specified through three attributes:

  • Error Source: The specific type of assertion affected by the error.
  • Error Nature: The nature of the error, which can be syntactic or semantic.
  • Error Type: The source and nature of an error lead to a specific error type. For example, an instance assertion might contain a spelling mistake in the type name.

Knowledge Cleaning Approaches, techniques and dimensions

Knowledge cleaning approaches implement cleaning techniques that define how the processing during detection or correction works. These approaches and techniques, therefore, have different dimensions, i.e., characteristics. These dimensions represent, for example, which type of source is used, and how this source is processed. For example, the ontology distinguishes between two types of sources:

  • Internal Sources: The knowledge graph and its assertions.
  • External Sources: Background knowledge such as the Web, other databases, or structured documents.

Selection of approaches

Sources—whether internal or external—vary in quality, which includes aspects like completeness and correctness. Therefore, this ontology adopts concepts and relations from the Data Quality Vocabulary. The sources are subclasses of a data set, which is the target of quality measurements. Inside the visualization these concepts and relations use a different font-family. The quality specifications are required by certain dimensions of approaches, which enables the selection of approaches.

Application of Knowledge Cleaning Approaches

The ontology also represents the application process of selected approaches. This process involves:

  • Tasks, and Conditions.
  • Structuring a cleaning pipeline using application algorithms.
  • Ensuring correctness of predicted results through additional validation and error detection processes.

Central and Subordinate Questions

The ontology is designed to answer one main question:

How to select and apply knowledge cleaning approaches, considering a knowledge graph, user requirements, and available sources?
It also supports subordinate questions, such as:
  • Which approach targets *** error?
  • Which approaches have *** dimension?
  • Which cleaning techniques support *** background knowledge?
  • What type of background knowledge does technique *** use?
  • What are the quality specifications for *** type of background knowledge?

Conclusion

By providing well-defined semantics for errors, approaches, techniques, quality specifications, and application algorithms, the AKCO enables a structured and systematic knowledge cleaning process. It ensures error detection, correction, and result validation within knowledge graphs.

HKCO: Description back to ToC

The anatomy of knowledge cleaning ontology represents important features of knowledge cleaning. These features include knowledge cleaning approaches, techniques and backgroundknowledge. It aims to provide the context from which knowledge cleaning can be performed. The ontology proviedes means to select approaches for a custom cleaning process, depending on for example, the knowledge graph and available additional cleaning sources. After the selection the approaches need to be applied, this application is represented by the Application algorithm and logic classes.
